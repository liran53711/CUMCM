# 三人组赛题选择策略

## 队伍配置

当前队伍为三名南大大二本科生：

- 软件工程：主程序、数据处理、模型实现、可视化、支撑材料复现。
- 集成电路：主建模、公式推导、优化/仿真、工程机理分析。
- 数字经济：主论文、指标体系、现实背景解释、管理/经济类问题理解。

整体评价：配置均衡，适合 B/C 题，也可以在 A 题背景可理解时尝试 A 题。

## A/B/C 题适配度

| 题号 | 适配度 | 原因 |
| --- | --- | --- |
| A 题 | 中等 | 集成电路同学有工程数学优势，但新手队容易被硬机理和推导拖慢 |
| B 题 | 较高 | 通常综合性强，三个人都能发挥，适合作为首选观察对象 |
| C 题 | 较高 | 数据分析和现实解释占比高，软件工程与数字经济优势明显 |

默认策略：优先认真评估 B/C；A 题若题面清楚、附件友好、机理不太硬，可纳入候选。

## 开赛后 2 小时流程

### 0-30 分钟：快速浏览

三人各自快速读 A/B/C 题，记录：

- 题目背景
- 附件类型
- 每一问要做什么
- 初步模型方向
- 最大风险

### 30-75 分钟：分工深读

- 软件工程：检查附件，判断数据能否读取、清洗和建模。
- 集成电路：判断数学结构，是否有目标函数、约束、方程或仿真机制。
- 数字经济：判断背景是否能讲清楚，指标体系和论文叙事是否自然。

### 75-105 分钟：打分比较

每题按 5 项评分，每项 1-5 分：

| 指标 | 说明 |
| --- | --- |
| 可启动性 | 第一问能否快速做出结果 |
| 数据可用性 | 附件是否清晰、可读、可处理 |
| 模型可落地 | 是否能用 Python 在当天跑出 baseline |
| 论文可解释 | 结果是否容易讲清楚，有现实意义 |
| 风险可控 | 背景、公式、数据、时间风险是否可控 |

总分不是唯一标准。如果某题存在致命风险，即使总分高也不选。

### 105-120 分钟：最终定题

必须确定一个题，并立刻拆任务：

- 软件工程：建立代码目录，读取数据，做 EDA 和 baseline。
- 集成电路：写变量、假设、模型框架和求解流程。
- 数字经济：搭论文目录，写问题重述、背景、假设和摘要草稿。

## 选题红线

出现以下情况要谨慎：

- 第一问都不知道如何下手。
- 附件数据打不开或含义不清，且无法在短时间内修复。
- 需要大量专业背景才能理解题意。
- 模型看似高级，但无法解释结果。
- 只能写代码，写不出数学模型。
- 只能写模型，跑不出任何数值结果。

## 适合本队的模型路线

### B/C 题常用组合

1. 数据清洗 + 描述统计 + 可视化
2. 指标体系或特征工程
3. 基础模型 baseline
4. 改进模型
5. 稳健性/敏感性分析
6. 结果解释和建议

### 可优先掌握的工具

- Python：`pandas`, `numpy`, `matplotlib`, `scipy`, `statsmodels`, `sklearn`
- 优化：`scipy.optimize`, `pyomo`
- 图论：`networkx`
- 论文：Word 或 LaTeX 均可，但必须提前统一模板

## 三人协作原则

- 软件工程同学不能只做代码，要参与模型选择。
- 集成电路同学不能只推公式，要保证模型能算。
- 数字经济同学不能只润色文字，要负责指标体系和现实解释。
- 所有图表必须服务论文结论，不能为了堆图而画图。
- 第一天晚上必须有能运行的 baseline 和论文骨架。
